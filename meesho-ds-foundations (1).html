Supplier Panel

Supplier Dashboard

B2B dashboard for sellers managing listings, orders, payments, and logistics. Brand: Crystal Indigo — trust and authority in a data-dense environment.

Key design decision

Primitives encode raw values (e.g. Crystal Indigo/600 = #4c31b9). Semantics encode role-based aliases by property — bg / text / border / icon / focus / overlay. Disabled states always map to Neutral/200 (#d6d6db). On-base text always maps to White. No dark mode in v1.

Key design decision

Primitives are named by their pixel value (space-8 = 8px). Designers use semantic tokens in Figma; developers reference primitives in code. MWeb platform mode steps semantic tokens down one size. Touch targets step up. Both via Figma Variable modes — no separate token set needed.

Key design decision

Display can only be used in Landing Pages / First Call sections. Headings can have nested content or stand alone. Body always stands alone — never nested. Mobile steps down uniformly; touch targets compensate via spacing tokens.

Valmo Panel

Valmo Dashboard

Same foundations as Supplier Dashboard. The single differentiator is brand color: Valmo Blue — a distinct identity within the same system architecture.

01
Foundation

Color — Brand only differs

All 7 shared palettes are identical. Only the Brand palette changes — Valmo Blue replaces Crystal Indigo. Semantic token names are unchanged; only the brand/* primitive resolves to a different hex.

Valmo Blue ✦ changed100–900

How panel theming works

One Figma Variable mode swap — only brand/* primitives resolve to a different hex. All semantic tokens, spacing, radius, and type remain in the same file, unchanged. Solid palettes are new; dashed palettes are shared with Supplier.
